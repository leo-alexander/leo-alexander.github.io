Problem Solving

Here are some problem solving techniques and processes:

  • Pseudocode: Using plain english to describe each line of code, so that you can easily target suspected areas where it may have gone wrong. This is the first technique I use when confronting a problem.
  • Try Something: Even if I suspect that something isn't going to work as I build it, I'll build it anyway so I have something to work off when I come across a problem. The hard part is when it works and you don't know why.
  • Rubber Ducky Method: Essentially the same as pseudocode, if you need to explain your code to a physical object instead of in your head.
  • Reading Error Messages: Can be very useful.
  • Console.log: Adding a few console.log outputs allows you to check results at different stages.
  • Google: Im currently learning what thousands of people have learnt before, so there is bound to be help online. Googling also unearths databases/wikis based on each language, which often become my go-tos.
  • Asking your peers for help: At my previous coding job in London I learnt that asking my more experienced coworkers for help was quite often a quicker solution.
  • Asking you coachers: My boss was my backup if my coworkers couldn't help. Often it would lead to bigger problems being discovered and more than just me learning something.
  • Reflection: Once I have solved a problem, I will keep the method and solution in the back of my mind for when a similar problem unearths itself later on.

  var newArray = array.map(function(i) {
  return i * 10; // newArray === [2,4,6]
});
.filter() Similar to .map(), it filters an array based on a boolean function. 
var array = [1,2,3];
  function big (i) {
    return i > 2;}
  var newArray = array.filter(big);
  //newArray === [3]
