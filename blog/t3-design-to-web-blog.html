Responsive Design

Responsive Design is the practise of designing a website so that it adapts to certain variables, most importantly that of screen size. This is important especially now with the use of mobile phones and tablets being higher than that of desktop computers, meaning screen sizes are extremely diverse. Responsive design allows all users access to a fully functioning website

In most cases, it is considered best practise to start the design stage with a mobile sized screen. This reduces the risk of not being able to neatly present all information on a small screen after designing for a larger screen.

CSS Frameworks like Bootstrap and Skeleton are downloadable code presets that aim to reduce development time by covering the important basics. Pro: Clean and fast. Con: It's only faster if the developer is used to it.

The decision to have the contact details section span the entire width of the page as opposed to the sidebar (I figured that there is not much content as of now so the empty space should be filled), and the removal of images on blog links for aesthetic purposes.

Other changes were due to technical reasons. I initally planned to continue using Google's Material Design, using "cards" for blog posts as I had done previously. I found it difficult to implement in addition to using the Skeleton framework. I'm sure they can work together - I just need to take some time later on to work it out.
