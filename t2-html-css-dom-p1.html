HTML, CSS, DOM, and Boxes

To use and analogy, HTML, CSS and DOM could be seen as an Urban Planner's regional strategic plan. The HTML establishes the different geographical areas to be zoned (ie. each div is a region that needs zoning rules). CSS is a list of zone types. Some regions will be zoned residential, others commercial, and others industrial. The browser then uses the Document Object Model to apply the zoning rules (CSS) to each zone (HTML).

HTML works in boxes. Even non-rectangular content on web browsers are secretly boxes. "Boxifying Content Design" is the practise of recognising this and designing the webpage as though it is split into boxes, then styling them afterwards. Its the most straight forward way of building a website.

I still use this site as a guide when building a website. Its super useful for getting my head around the box-model, which explains the structure of each element in an html file. CSS can target each part of the box - margin, border, padding, and content - so its easier to work with boxes.
